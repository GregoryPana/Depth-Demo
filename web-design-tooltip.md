Web Design Tool tips

## Core idea

You only need four main **skills** to become a top‑tier web designer: typography, layout, color, basic coding, and conversion-focused design, rather than trying to learn every possible tool or trend.

---

## 1. Typography skills

- Text takes up most of a website’s real estate, and most designers can feel when typography is off but cannot explain why.
- Mastery comes down to two questions: how to pick fonts, and how to set them up with a type scale system so text is cohesive, predictable, and intentional.

### Choosing fonts

- Avoid overused free fonts so your designs don’t blend in; use alternative sources like Fontshare (clean, high‑quality, free) and Uncut (more experimental).

### Setting up a type scale

- Use a type scale system to define font sizes, letter spacing, and line height so you’re not eyeballing everything.
- Start with a base paragraph size around 16px, then scale headings using a ratio like the “Major Third” (about 25% steps: 16 → 20 → 25 → … up to H1), preferably using rem units.
- Letter spacing: leave body text at default, but slightly tighten spacing for headings as they get larger.
- Line height: use about 150% of the font size for paragraphs (e.g., 16px × 1.5), and tighten it as text gets larger for more impact and scannability.
- You can generate a full type scale quickly with tools like type-scale.net by choosing a base size and ratio.

## 2. Layout skills

- Pros do not guess layout; they rely on systems: a grid, a spacing system, and visual hierarchy.

### Grid system

- Recommended grids: 12 columns on desktop, 8 on tablet, and 4 on mobile, because they divide easily into 1–4 column layouts.

### Spacing system

- Use an 8‑point grid (spacing always multiples of 8: 8, 16, 24, 32, etc.) to give designs rhythm and breathing room.
- This approach is used by systems such as Google’s Material Design and Apple’s design systems.

### Visual hierarchy

- People scan, not read, so layout must guide the eye from most important to least important content.
- Key principles:
    - Proximity: keep related items close together.
    - Size: use larger elements for more important content.
    - Contrast: use size, weight, or color (e.g., different opacity for headers vs body text) to signal importance.
    - Alignment: clean lines and consistent alignment create clear structure.
- Using these principles together makes layouts feel effortless to navigate and separates pros from beginners.

## 3. Color skills

- Many designers feel overwhelmed by the huge number of colors, but pros focus on intentional, limited color use.
- Limiting the color palette to 2–3 colors per project, each with a specific role, is enough for most work.

### 60–30–10 rule

- 60% neutral colors (whites, blacks, grays) for background and text.
- 30% secondary colors for backgrounds, cards, headers, or visuals.
- 10% accent color for elements that must stand out, like buttons and CTAs.
- As your confidence grows, you can expand the palette, but it’s not necessary.

### Opacity and reuse

- Instead of adding more colors, derive variations by adjusting opacity of your primary color to create layers.
- This technique is visible in Google’s Material Design, where opacity creates many color variations.
- One well‑used color is better than many random colors.

### Contrast and accessibility

- Color combinations must be readable, not just pretty, so always check color contrast.
- Recommended minimum contrast ratios: 3:1 for large text and 4.5:1 for smaller text.

### Borrowing palettes

- Most great palettes are borrowed and refined rather than invented from scratch.
- You can inspect any site’s palette using Chrome’s “CSS Overview” tool: open DevTools, go to More tools → CSS Overview, capture the overview, and you’ll see colors, fonts, and type scale.

## 4. Conversion skills

- The fourth skill is **conversion**: designing for action instead of just looks.

### One clear goal per page

- Each page should have a single primary goal, such as getting a product purchase, a booked call, or a lead magnet signup.
- Trying to accomplish too many things at once causes user drop‑off.

### Strategic CTAs

- Calls‑to‑action must be clear and visible within seconds of page load.
- One advice is to place CTAs in the hero section, in the navigation, and then every 2–3 “screenfuls” as the user scrolls.

### Trust, clarity, and emotion

- Conversion design depends on trust, clarity, and especially feelings; people buy from brands they know, like, trust, and emotionally connect with.
- Sites should speak to real motivations, clearly explain what users get, and show they are not alone, using elements like testimonials, reviews, and social proof.
